# 快速开始指南

> 适用对象：首次使用本工作流的用户  
> 更新时间：2026-04-13（中文化增强版）

---

## 30 秒快速上手

### 场景：你想分析"CRM 系统"的竞品

**步骤 1：触发角色 1（战略规划）**
```
/角色 1 CRM 系统
```
等待 1-2 分钟，获得 3 个竞品分析报告。

**步骤 2：触发角色 2（部署与截图）**
```
/角色 2
```
等待 5-10 分钟，获得可运行的竞品部署 + 功能截图。

**步骤 3：触发角色 3（PRD 生成）**
```
/角色 3
```
等待 2-3 分钟，获得中文 PRD 文档。

**步骤 4：触发角色 4（原型生成）**
```
/角色 4
```
等待 5-10 分钟，获得全中文可交互 HTML 原型。

**完成！** 你将获得：
- ✅ 3 个热门竞品分析（含 SWOT）
- ✅ 可运行的竞品部署 + 功能截图
- ✅ 中文 PRD 文档
- ✅ 全中文可交互 HTML 原型

## 完整工作流

```
用户输入产品类型
       ↓
┌─────────────────────────────────────────────────┐
│ 角色 1：战略规划工程师                           │
│ - GitHub 搜索热门前 3                            │
│ - SWOT 分析 + 亮点功能标注                        │
│ - 推荐最适合深入分析的竞品                      │
└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│ 角色 2：部署与截图工程师                         │
│ - 自动部署竞品（Docker/npm）                     │
│ - 页面遍历 + 截图                                │
│ - 识别功能模块清单                               │
│ - 标注亮点功能的实际表现                        │
└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│ 角色 3：需求文档工程师                           │
│ - 基于竞品分析和 SWOT                            │
│ - 借鉴亮点功能                                   │
│ - 输出中文 PRD 文档                              │
└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│ 角色 4：原型工程师                               │
│ - 基于 PRD 生成全中文可交互原型                   │

└─────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────┐
│ 人工审核                                         │

└─────────────────────────────────────────────────┘
```

## 中文化规范

从角色 3 开始，所有输出使用中文：

| 角色 | 中文化内容 |
|------|-----------|
| 角色 3 | PRD 文档（功能名/场景/交互流程） |
| 角色 4 | HTML 原型（UI 文案/表单字段/提示信息） |

**示例**：
- 按钮："确定" 而非 "OK"
- 表单："手机号" 而非 "Phone"
- 提示："加载中..." 而非 "Loading..."

## 常见问题

### Q1: 角色 1 输出的报告格式不对怎么办？
A: 检查是否按照 `/角色 1 [产品类型]` 的格式触发，确保产品类型是一个简短的短语。

### Q2: 角色 2 部署失败怎么办？
A: 角色 2 会尝试标准修复（换源、换端口），如果仍然失败会记录原因并跳过。查看部署报告中的"失败原因"部分。

### Q3: 如何查看之前项目的输出？
A: 所有输出保存在 `工作输出/` 目录下，按项目名分类。

### Q4: 原型真的是中文的吗？
A: 是的！角色 4 生成的 HTML 原型所有 UI 文案、表单字段、提示信息都使用中文。详见 [[01-工作流程/04-原型工程师/角色说明 \| 原型工程师角色说明]]。
