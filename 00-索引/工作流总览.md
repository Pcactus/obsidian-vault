## 工作流概述

本工作流将**2-3 周的竞品分析 + PRD+ 原型周期压缩到 4-8 小时**。

```
用户输入产品类型
       ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 1：竞品分析报告 (角色 1)                                  │
│  - GitHub 搜索热门前 3                                        │
│  - SWOT 分析 + 亮点功能标注                                   │
│  - 推荐最适合深入分析的竞品                                   │
└─────────────────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 2：产品使用手册 (角色 2 增强版)                            │
│  - 功能模块清单（树状结构）                                    │
│  - 每个功能的操作步骤 + 截图 + 交互逻辑                         │
│  - 亮点功能重点标注                                           │
│  - 覆盖率目标：核心功能 100%，边缘功能 80%+                     │
└─────────────────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 3：PRD 文档 (角色 3)                                      │
│  - 基于竞品分析和 SWOT                                        │
│  - 借鉴亮点功能                                               │
│  - 标准 PRD 格式（背景、目标用户、功能列表、优先级）              │
│  - 全中文输出（功能名/场景/交互流程）                          │
└─────────────────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段 4：可交互 HTML 原型 (角色 4)                               │
│  - 基于 PRD 生成                                               │
│  - 可点击交互                                                 │
│  - 可直接给前后端开发使用                                      │
│  - 全中文界面（UI 文案/表单字段/提示信息）                        │
└─────────────────────────────────────────────────────────────┘
       ↓
┌─────────────────────────────────────────────────────────────┐
│  人工审核环节                                                 │
│  - 审核 PRD 和原型是否符合业务需求                              │
│  - 确认后即可交付开发团队                                      │
└─────────────────────────────────────────────────────────────┘
```

## 人工审核环节

| 审核点 | 时机 | 审核内容 |
|--------|------|----------|
| 审核点 1 | 阶段 1 完成后 | 确认分析的竞品是否合理，是否有遗漏 |
| 审核点 2 | 阶段 2 完成后 | 对照本地部署的项目做功能验证 |
| 审核点 3 | 阶段 3-4 完成后 | 审核 PRD 和原型是否符合业务需求 |

## 技术架构

### 核心组件

| 组件 | 职责 | 技术选型 |
|------|------|---------|
| GitHub 搜索 | 搜索热门竞品 | GitHub API / Web 搜索 |
| 部署引擎 | 自动部署竞品 | Docker / npm / python |
| 页面遍历 | 自动点击、截图 | Puppeteer / Playwright |
| 功能识别 | 识别页面功能模块 | AI 视觉分析 + DOM 分析 |
| 文档生成 | 生成 SWOT、使用手册、PRD | LLM + 模板 |
| 原型生成 | 生成可交互 HTML | React/Vue + AI 生成 |

### 现有资产复用

|----------|------------|

| `/角色 2`（部署与截图工程师） | 自动部署 + 页面遍历 + 功能模块识别 |

## 截图规范

### 命名规则

| 场景 | 命名格式 | 示例 |
|------|----------|------|
| **部署报告** | `[序号]-[功能名].png` | `00-许可协议.png`、`01-系统检查.png` |
| **功能总览** | `[模块名]_[功能名].png` | `销售管理_线索管理.png` |
| **模块详情** | `[模块名]_[功能名].png` | `销售管理_创建线索.png` |

> 注：功能总览和模块详情的命名格式相同，但存放路径不同

### 存放路径

| 文档类型 | 截图存放路径 | 引用方式 |
|----------|-------------|---------|
| 部署报告 | `00-部署报告/assets/` | `![说明](./assets/00-功能名.png)` |
| 功能总览 | `01-功能模块总览/assets/[模块]/` | `![说明](./assets/模块/模块_功能.png)` |
| 模块详情 | `功能模块/[模块]/[功能]/assets/` | `![说明](./assets/模块_功能.png)` |

### 引用格式

**Markdown 图片语法**（推荐，兼容性好）：
```markdown
![图片说明](./assets/文件名.png)
```

**Obsidian Wiki-link 语法**（可选）：
```markdown
![[attachments/文件名.png]]
```
