# GStack 技能对话指南

> 适用对象：已经知道 `/` 技能，但不确定如何使用的用户  
> 更新时间：2026-04-09

---

## 核心问题：我需要每句话都用 `/技能名` 吗？

### 简短回答

**不需要！** `/` 只用于**启动技能**，启动后可以正常对话。

---

## 技能对话的生命周期

```
┌─────────────────────────────────────────────────────────┐
│  你：/角色 1 CRM 系统         ← 启动技能（需要 /）         │
│       ↓                                                 │
│  Claude: 好的，我开始执行角色 1...                       │
│          正在搜索 GitHub...                              │
│          [技能执行中]                                    │
│          ↓                                               │
│  你：能不能再多找一个竞品？     ← 继续对话（不需要 /）     │
│          ↓                                               │
│  Claude: 好的，我来扩展搜索...                           │
│          [技能继续执行]                                   │
│          ↓                                               │
│  Claude: 完成！报告已生成。       ← 技能执行结束          │
└─────────────────────────────────────────────────────────┘
       ↓
[回到普通对话模式]
       ↓
你：帮我写个邮件总结这个报告   ← 普通对话（不需要 /）
```

---

## 何时需要用 `/`？

| 场景 | 需要 `/` 吗？ | 示例 |
|------|----------|------|
| **启动新技能** | ✅ 需要 | `/角色 1 CRM 系统` |
| **技能执行中继续对话** | ❌ 不需要 | "再多找一个竞品" |
| **切换到另一个技能** | ✅ 需要 | `/角色 2` |
| **问普通问题** | ❌ 不需要 | "输出存在哪里？" |
| **让 Claude 修改内容** | ❌ 不需要 | "把 SWOT 表格放大一点" |
| **追问细节** | ❌ 不需要 | "第三个竞品的团队规模是多少？" |

## 如何知道当前在哪个技能中？

### 方法 1：看对话上下文

Claude 在技能执行时通常会说：

```
Claude: 好的，我开始执行 **角色 1** 技能...
       正在搜索 GitHub 热门项目...
```

或者输出格式有明显特征：

```
Claude: ## 竞品分析报告
       ### 竞品 1: Plane
       - SWOT 分析...
```

### 方法 2：直接问

```
你：我们现在在哪个技能中？

Claude: 当前正在执行角色 1（战略规划工程师）技能。
       需要切换到其他技能吗？
```

### 方法 3：看界面显示（如果有）

某些聊天界面会显示：
- 当前调用的工具
- 技能名称
- 执行状态

## 重要概念：技能不是"模式切换"

### 错误理解 ❌

```
/角色 1 = 进入"角色 1 模式"
         之后所有对话都在这个模式中
         需要用 /exit 退出
```

### 正确理解 ✅

```
/角色 1 = 一次性指令
         = "请用角色 1 的流程来做这件事"
         执行完就结束
         回到普通对话
```

## 完整对话示例

### 示例 1：单个技能执行

```
你：/角色 1 CRM 系统

Claude: 好的，我开始分析 CRM 系统的竞品...
       [执行搜索、分析、输出报告]
       完成！这是完整的竞品分析报告。

你：第三个竞品的 SWOT 能再详细一点吗？

Claude: 当然，我来扩展第三个竞品的 SWOT 分析...
       [继续完善]

你：好的，现在帮我写个邮件把这个报告发给团队

Claude: 好的，我来帮你起草邮件...
       [普通对话，不是技能]
```

---

### 示例 2：连续使用多个技能

```
你：/角色 1 CRM 系统

Claude: [执行角色 1，输出竞品分析报告]
       完成！

你：/角色 2

Claude: 好的，我现在执行角色 2...
       [开始部署竞品并截图]
       部署完成！访问地址：http://localhost:3000

你：截图能保存到另一个文件夹吗？

Claude: 当然，我来调整输出路径...
       [继续执行角色 2 的剩余部分]
```

---

### 示例 3：技能执行中被中断

```
你：/角色 1 CRM 系统

Claude: 好的，我开始搜索 GitHub 热门 CRM 项目...
       找到 3 个竞品：Plane, Saleor, Odoo
       正在进行 SWOT 分析...

你：等一下，我改主意了，先分析任务管理工具吧

Claude: 好的，那我们用任务管理工具重新来。
       你可以输入 `/角色 1 任务管理工具` 来重新启动。

你：/角色 1 任务管理工具

Claude: 好的，我开始分析任务管理工具的竞品...
```

## 技能切换的最佳实践

### ✅ 推荐做法

```
你：/角色 1 CRM 系统
Claude: [执行完成]

你：好的，这个分析很好。现在帮我部署。
Claude: 你可以用 `/角色 2` 来自动部署。

你：/角色 2
Claude: [开始部署]
```

### ⚠️ 不推荐做法

```
你：/角色 1 CRM 系统
Claude: [执行中...]

你：/角色 2          ← 技能 1 还没执行完就切换
Claude: [困惑，需要澄清]
```

**建议**：等当前技能执行完成（Claude 说"完成"或类似的话）再切换到另一个技能。

## 常见问题

### Q1: 技能执行到一半，我想换个方向怎么办？

A: 直接告诉 Claude 你的新需求，它会根据情况决定：
- 继续当前技能（如果可以调整）
- 建议你用另一个技能

```
你：/角色 1 CRM 系统
Claude: [执行中...]

你：其实我更想看开源的项目，商业的不要

Claude: 好的，我来调整搜索条件，只关注开源项目...
```

---

### Q2: 我可以用 `/` 中断当前技能吗？

A: 可以，但建议先说明：

```
你：/角色 1 CRM 系统
Claude: [执行中...]

你：先停一下，我有个问题

Claude: 好的，什么问题？

你：/角色 2

Claude: 好的，切换到角色 2...
```

---

### Q3: 技能执行完成后，我还需要做什么吗？

A: 不需要！技能执行完成后自动回到普通对话。你可以：
- 继续追问
- 切换到另一个技能
- 结束对话

---

### Q4: 如何查看当前可用的技能列表？

A: 输入 `/` 后不要按回车，系统会弹出可用技能列表。

或者查看：[[GStack 技能]] 文档

## 总结

```
┌────────────────────────────────────────────────────┐
│                   技能对话流程                      │
├────────────────────────────────────────────────────┤
│                                                    │
│  1. 用 /技能名 启动                                 │
│     ↓                                              │
│  2. 技能执行中（正常对话，不需要 /）                │
│     ↓                                              │
│  3. 技能执行完成                                    │
│     ↓                                              │
│  4. 回到普通对话                                    │
│     ↓                                              │
│  5. 需要时用 / 启动另一个技能                       │
│                                                    │
└────────────────────────────────────────────────────┘
```

**核心原则**：
- `/` = 启动技能，不是进入模式
- 技能执行中 = 正常对话
- 技能完成后 = 自动回到普通对话
- 不确定时 = 直接问"我们现在在哪个技能中？"
