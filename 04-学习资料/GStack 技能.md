# GStack 技能傻瓜式教程

> 适用对象：第一次使用 GStack 技能的用户  
> 更新时间：2026-04-09

---

## 什么是 GStack 技能？

GStack 技能是一组**预定义的快捷指令**，让你可以用简单的命令触发复杂的工作流。

**类比**：就像游戏的"宏命令"，一键执行一连串操作。

---

## 核心问题：如果不用 `/` 命令，Claude 会自己调用技能吗？

### 简短回答

**不会自动调用**。`/` 技能是你**主动触发**的快捷方式。

### 详细解释

GStack 技能实际上是存储在 `~/.claude/skills/gstack/[技能名]/SKILL.md` 的配置文件。当你输入 `/技能名` 时：

1. Claude 读取对应的 `SKILL.md` 文件
2. 按照文件中定义的步骤执行
3. 包括前置检查、工具调用限制、钩子脚本等

**如果你不用 `/` 命令**：
- Claude 仍然可以帮你完成任务
- 但不会按照技能的标准化流程执行
- 输出格式可能不一致，可能遗漏某些步骤

---

## 三种调用方式对比

### 方式 1：使用 `/技能名` 命令（推荐）

```
/角色 1 CRM 系统
```

**发生了什么**：
1. 你明确告诉 Claude："用角色 1 的技能来分析 CRM 系统"
2. Claude 读取技能配置文件（`~/.claude/skills/gstack/角色 1.md`）
3. 按照技能中定义的步骤执行

**优点**：
- ✅ 明确意图，Claude 知道你要做什么
- ✅ 技能内部有预设的最佳实践
- ✅ 输出格式一致

---

### 方式 2：用自然语言描述（Claude 可能建议使用技能）

```
帮我分析一下 CRM 系统的竞品
```

**发生了什么**：
1. Claude 理解你的需求
2. Claude **可能**会建议你使用 `/角色 1`
3. 但 Claude 也可以直接开始工作（不调用技能）

**示例对话**：
```
你：帮我分析一下 CRM 系统的竞品

Claude: 好的，我可以用 `/角色 1 CRM 系统` 来帮你进行竞品分析。
       这个技能会：
       1. 搜索 GitHub 上最热门的 3 个 CRM 竞品
       2. 进行 SWOT 分析
       3. 标注亮点功能
       4. 推荐最适合深入分析的竞品

       要我现在执行吗？
```

---

### 方式 3：完全不用技能（纯手动）

```
你：帮我找几个 CRM 系统的竞品

Claude: 好的，我来帮你搜索...
       （开始手动搜索，没有标准化流程）
```

**会发生什么**：
- ✅ Claude 仍然可以帮你完成任务
- ⚠️ 没有技能的标准化流程，可能遗漏步骤
- ⚠️ 输出格式可能每次都不一样

## 技能文件长什么样？

每个技能都有一个 `SKILL.md` 文件，包含：

```markdown
---
name: investigate
description: 系统化问题调查
allowed-tools:
  - Bash
  - Read
  - Write
  - Grep
hooks:
  PreToolUse:
    - matcher: "Edit"
      hooks:
        - type: command
          command: "bash check-freeze.sh"
---

## Preamble（前置脚本）
...bash 代码，运行环境检查...

## 主体指令
...告诉 Claude 如何执行任务...
```

**关键部分**：
| 部分 | 作用 |
|------|------|
| `allowed-tools` | 限制技能可以使用的工具 |
| `hooks` | 在执行某些操作前运行脚本（如检查编辑边界） |
| `Preamble` | 前置 bash 脚本，检查环境、版本等 |
| 主体指令 | 告诉 Claude 具体如何执行任务 |

## 基础用法

### 如何触发技能？

在对话框中输入 `/` 开头的内容，系统会自动联想可用技能：

```
/角色 1 CRM 系统     ← 触发"战略规划工程师"角色
/browse https://...  ← 触发"网页浏览"技能
/review              ← 触发"代码审查"技能
```

### 技能格式

```
/技能名 [参数]
```

| 部分 | 说明 | 示例 |
|------|------|------|
| `/技能名` | 必须的，斜杠 + 技能名 | `/角色 1` |
| `[参数]` | 可选的，技能需要的输入 | `CRM 系统` |

## 安全技能（重要！）

> [!warning] 危险操作警告
> 以下技能用于保护你免受误操作伤害：

| 技能 | 作用 |
|------|------|
| `/careful` | 执行危险命令前警告你（如 `rm -rf`） |
| `/guard` | 完全安全模式：警告 + 限制编辑目录 |
| `/freeze [目录]` | 锁定编辑范围，只能编辑指定目录 |
| `/unfreeze` | 解除编辑限制 |

**示例**：
```
/freeze 02-工作输出/CRM   ← 只能编辑这个目录
/角色 1 CRM 系统          ← 安全，在允许范围内
/角色 1 新產品            ← 警告，可能输出到其他目录
/unfreeze                 ← 解除限制
```

## 技能组合技

某些场景需要组合使用多个技能：

### 完整工作流

```
/角色 1 [产品类型]  →  /角色 2  →  /review  →  /land-and-deploy
     ↓                    ↓           ↓              ↓
  竞品分析           部署截图    代码审查      部署上线
```

### 调试流程

```
/investigate  →  找到问题  →  修复  →  /qa  →  验证修复
```

## 常见问题

### Q1: 技能不响应怎么办？
A: 检查技能名是否拼写正确，确认斜杠 `/` 是英文符号。

### Q2: 如何查看所有可用技能？
A: 输入 `/` 后不要按回车，系统会弹出技能列表。

### Q3: 技能执行到一半卡住了？
A: 输入 `/stop` 或 `Ctrl+C` 中断，然后重新触发。

### Q4: 如何更新技能？
A: 运行 `/gstack-upgrade` 检查并更新到最新版。
