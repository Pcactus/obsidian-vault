## 1. 产品概述

### 1.1 产品定位
面向个人/小团队的轻量化线索管理工具，借鉴 SuiteCRM 的功能完整性和 Twenty 的现代化体验，提供简洁高效的潜在客户管理解决方案。

### 1.2 目标用户
| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| 销售人员 | 个人或小团队销售 | 快速录入线索、跟踪跟进状态、conversion 转化 |
| 销售主管 | 小团队管理者 (1-10 人) | 线索分配、团队线索概况、批量操作 |
| 市场人员 | 负责线索获取 | 批量导入线索、线索来源分析 |

### 1.3 核心价值
解决小团队在客户获取阶段的核心痛点：**线索分散、跟进混乱、转化丢失**，提供一站式的线索获取→跟进→转化的完整管理流程。

### 1.4 差异化优势
| 竞品       | 借鉴点                  | 差异化改进                   |
| -------- | -------------------- | ----------------------- |
| SuiteCRM | 完整的线索 CRUD、状态流转、转换机制 | 简化地址字段、移除传真等冗余字段，适配国内场景 |
| Twenty   | 现代化 UI、多视图切换、自定义字段   | 保持简洁，初期不做复杂的自定义对象系统     |
| Krayin   | 模块化架构设计              | 单体应用优先，后期支持模块化扩展        |

### 2.2 功能详解

#### 模块 1：线索列表

##### 线索列表展示 - P0
- **功能描述**：以表格形式展示所有线索，支持列排序和基本信息查看
- **用户场景**：销售人员每天打开系统查看自己的线索列表，了解待跟进客户
- **功能点**：
  - 表格展示线索核心字段（姓名、公司、状态、来源、负责人、创建时间）
  - 点击列标题可切换升序/降序排序
  - 点击线索姓名跳转到详情页
  - 支持分页，每页默认 20 条
- **交互流程**：
  ```
  1. 用户进入线索模块
     → 系统加载线索列表（默认按创建时间倒序）
  2. 用户点击列标题
     → 系统切换排序方向（升序↔降序）
  3. 用户点击线索姓名
     → 系统打开线索详情页
  4. 用户点击分页器
     → 系统加载对应页数据
  ```
- **数据结构**：
  ```
  LeadListResponse:
    - items: Lead[]
    - total: number
    - page: number
    - pageSize: number
    - totalPages: number
  
  Lead:
    - id: string
    - firstName: string        // 姓
    - lastName: string         // 名
    - company: string          // 公司
    - status: enum             // 状态
    - source: enum             // 来源
    - assignedTo: User         // 负责人
    - createdAt: datetime
  ```
- **备注**：初期仅展示核心字段，避免列表过宽

##### 搜索与筛选 - P0
- **功能描述**：支持多维度搜索和筛选线索
- **用户场景**：销售人员需要快速找到特定线索，如"所有状态为新建的线索"
- **功能点**：
  - 关键词搜索（支持姓名、公司名称）
  - 状态筛选（新建/已联系/有意向/已评估/已转换/已关闭）
  - 来源筛选（网站/电话/推荐/活动/其他）
  - 负责人筛选（我的线索/我负责的/全团队）
  - 筛选条件改变时列表自动刷新
- **交互流程**：
  ```
  1. 用户在搜索框输入关键词
     → 系统实时显示匹配结果（防抖 300ms）
  2. 用户选择筛选条件
     → 系统自动刷新列表，URL 同步筛选参数
  3. 用户点击清除筛选
     → 系统重置所有筛选条件，恢复默认列表
  ```
- **备注**：借鉴 Twenty 的筛选器设计，支持筛选条件保存在 URL 中方便分享

##### 批量操作 - P1
- **功能描述**：对多条线索进行批量分配、删除、导出操作
- **用户场景**：销售主管需要将一批新线索分配给团队成员
- **功能点**：
  - 勾选多条线索
  - 批量分配给指定负责人
  - 批量删除（软删除，弹出确认框）
  - 批量导出为 CSV
- **交互流程**：
  ```
  1. 用户勾选多条线索
     → 系统激活批量操作按钮，显示已选数量
  2. 用户点击"批量分配"
     → 系统弹出负责人选择器
  3. 用户选择目标负责人并确认
     → 系统执行批量分配，显示操作结果
  4. 用户点击"批量删除"
     → 系统弹出二次确认框
     → 用户确认后执行软删除
  ```
- **备注**：批量删除需要二次确认，防止误操作

#### 模块 2：创建线索

##### 创建线索表单 - P0
- **功能描述**：录入新的潜在客户信息
- **用户场景**：销售人员接到新客户电话咨询后，需要录入系统
- **功能点**：
  - 基本信息：姓、名、职位、部门
  - 联系方式：主要邮箱、办公电话、手机
  - 公司信息：公司名称、所属行业、公司网址
  - 线索信息：线索来源、线索状态、线索评级、负责人
  - 地址信息：省、市、区、街道（简化为单行地址）
  - 描述：备注信息
- **交互流程**：
  ```
  1. 用户点击"创建线索"按钮
     → 系统打开创建表单（弹窗或新页面）
  2. 用户填写表单
     → 系统实时验证格式（邮箱/电话/URL）
     → 必填字段为空时，保存按钮置灰
  3. 用户点击"保存"
     → 系统创建线索，跳转到详情页
  4. 用户点击"取消"
     → 系统返回列表页
  ```
- **数据结构**：
  ```
  LeadCreateRequest:
    // 必填
    - firstName: string (max 50)
    - lastName: string (max 50)
    
    // 可选
    - title: string (max 50)
    - department: string (max 50)
    - email: string (email format)
    - phone: string (max 20)
    - mobile: string (max 20)
    - company: string (max 100)
    - industry: enum
    - website: string (url format)
    - source: enum
    - status: enum (default: 新建)
    - rating: enum (Hot/Warm/Cold)
    - assignedUserId: string (default: current user)
    - address: string (single line)
    - description: string
  ```
- **备注**：
  - 简化 SuiteCRM 的地址字段（移除省市区拆分）
  - 移除传真字段（国内场景不适用）

##### 字段验证 - P0
- **功能描述**：对表单字段进行实时格式验证
- **用户场景**：用户输入邮箱时，系统即时提示格式是否正确
- **功能点**：
  - 邮箱格式验证（正则匹配）
  - 电话格式验证（支持国内外格式）
  - URL 格式验证
  - 必填字段非空验证
  - 字符长度限制提示
- **交互流程**：
  ```
  1. 用户输入邮箱
     → 系统失焦时验证格式
     → 格式错误时显示红色边框 + 错误提示
  2. 用户输入超长文本
     → 系统显示剩余字符数
  ```
- **备注**：借鉴 Twenty 的实时验证体验

#### 模块 3：线索详情

##### 线索详情展示 - P0
- **功能描述**：展示线索完整信息、关联记录和活动历史
- **用户场景**：销售人员在跟进客户前，查看客户完整背景信息
- **功能点**：
  - 基本信息展示区（所有字段只读展示）
  - 顶部操作按钮：编辑、转换、复制、删除
  - 关联记录：关联客户、联系人、商机、工单
  - 活动历史：系统日志（后期扩展邮件/电话/会议）
- **交互流程**：
  ```
  1. 用户从列表点击线索姓名
     → 系统加载详情页
  2. 用户点击"编辑"按钮
     → 系统打开编辑表单
  3. 用户点击"转换"按钮
     → 系统打开转换向导
  ```
- **页面布局**：
  ```
  ┌─────────────────────────────────────┐
  │ [基本信息]          [编辑][转换]     │
  ├─────────────────────────────────────┤
  │ 联系方式 | 公司信息 | 线索信息        │
  ├─────────────────────────────────────┤
  │ 关联记录                              │
  │ - 客户 - 联系人 - 商机 - 工单          │
  ├─────────────────────────────────────┤
  │ 活动历史                              │
  │ - 系统日志                            │
  └─────────────────────────────────────┘
  ```
- **备注**：初期仅展示系统日志，后期集成邮件/电话/会议记录

##### 编辑线索 - P0
- **功能描述**：更新线索的任意字段信息
- **用户场景**：客户信息变更后，销售人员更新系统记录
- **功能点**：
  - 全字段可编辑
  - 表单同创建表单
  - 修改后自动记录修改时间
- **交互流程**：
  ```
  1. 用户点击"编辑"按钮
     → 系统打开编辑表单（预填充当前数据）
  2. 用户修改字段
     → 系统实时验证
  3. 用户点击"保存"
     → 系统更新数据，返回详情页
  ```
- **备注**：与创建表单共用组件

##### 线索转换 - P1
- **功能描述**：将线索转为正式客户、联系人和商机
- **用户场景**：客户确认购买意向后，销售将线索转为正式客户
- **功能点**：
  - 转为客户（必选）
  - 转为联系人（可选）
  - 转为商机（可选）
  - 选择原线索处理方式（删除/保留）
  - 数据自动映射（姓名→客户名，公司→客户公司等）
- **交互流程**：
  ```
  1. 用户点击"转换"按钮
     → 系统弹出转换向导
  2. 用户勾选转换目标：
     ☑ 转为客户（必选，不可取消）
     ☐ 转为联系人
     ☐ 转为商机
  3. 用户确认数据映射预览
  4. 用户选择原线索处理方式：
     ○ 保留线索（状态变更为"已转换"）
     ○ 删除线索
  5. 用户点击"确认转换"
     → 系统创建客户/联系人/商机
     → 系统更新原线索状态
     → 跳转到新客户详情页
  ```
- **数据结构**：
  ```
  LeadConvertRequest:
    - convertToCustomer: boolean (required, true)
    - convertToContact: boolean (optional)
    - convertToOpportunity: boolean (optional)
    - keepOriginalLead: enum (keep/delete)
  
  LeadConvertResponse:
    - customerId: string
    - contactId?: string
    - opportunityId?: string
  ```
- **备注**：借鉴 SuiteCRM 的转换流程，初期商机信息简化

#### 模块 4：导入导出

##### CSV 导入 - P1
- **功能描述**：从 CSV 文件批量导入线索数据
- **用户场景**：市场人员参加活动后，需要批量导入收集的线索
- **功能点**：
  - 上传 CSV 文件
  - 字段映射（文件列 → 系统字段）
  - 数据预览
  - 导入结果报告（成功/失败数量及原因）
- **交互流程**：
  ```
  1. 用户点击"导入"按钮
     → 系统打开导入向导
  2. 用户上传 CSV 文件
     → 系统解析文件
  3. 用户映射字段
     → 系统拖拽匹配或自动匹配
  4. 用户预览数据
     → 系统展示前 10 条预览
  5. 用户确认导入
     → 系统执行批量导入
     → 系统显示导入报告
  ```
- **备注**：参考 SuiteCRM 的导入流程

##### CSV 导出 - P1
- **功能描述**：导出线索数据为 CSV 格式
- **用户场景**：销售人员需要备份数据或导出到 Excel 分析
- **功能点**：
  - 选择导出字段
  - 选择导出范围（全部/当前筛选结果/已勾选）
  - 生成 CSV 文件下载
- **交互流程**：
  ```
  1. 用户勾选线索或设置筛选条件
  2. 用户点击"导出"按钮
     → 系统弹出字段选择器
  3. 用户选择导出字段
  4. 用户确认导出
     → 系统生成 CSV 并触发下载
  ```

##### Excel 导入 - P2
- **功能描述**：支持 Excel (.xlsx) 格式直接导入
- **用户场景**：国内用户习惯使用 Excel，避免转换 CSV 的麻烦
- **功能点**：同 CSV 导入，额外支持 Excel 格式
- **备注**：增强功能，适配国内用户习惯

## 3. 非功能性需求

### 3.1 性能需求
- 列表页加载时间 < 2 秒（1000 条数据内）
- 搜索响应时间 < 500ms
- 支持 10+ 并发用户

### 3.2 安全需求
- 用户登录认证
- 基于角色的权限控制（查看/编辑/分配权限分离）
- 敏感操作二次确认（删除、转换）
- 软删除机制（可恢复）

### 3.3 兼容性需求
- 浏览器：支持 Chrome、Firefox、Safari、Edge（最新版本）
- 响应式设计：支持平板和手机端访问
- 数据接口：RESTful API 设计规范
