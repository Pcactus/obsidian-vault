## 核心交互流程

### 线索管理完整流程

```
1. 登录系统
   → 输入账号密码（admin/admin123）
   → 进入工作台

2. 查看线索列表
   → 从左侧边栏点击"销售管理 → 线索管理"
   → 查看线索表格，支持搜索、筛选

3. 创建线索
   → 点击列表页右上角"+ 新建线索"
   → 填写表单（必填：姓、名）
   → 点击"保存"，返回列表页

4. 查看线索详情
   → 点击列表中线索姓名
   → 查看完整信息

5. 编辑线索
   → 从详情页点击"编辑"按钮，或从列表页点击"编辑"
   → 修改字段后点击"保存"

6. 线索转换
   → 从详情页点击"转换"按钮
   → 选择转换目标（客户必选，联系人/商机可选）
   → 确认转换

7. 导入线索
   → 从列表页点击"导入"按钮
   → 上传 CSV/Excel 文件
   → 字段映射
   → 预览确认
   → 完成导入
```

### 批量操作流程

```
1. 在线索列表页勾选多条线索
   → 底部出现批量操作栏

2. 选择批量操作类型：
   - 分配：选择负责人后确认
   - 导出：直接导出选中线索为 CSV
   - 删除：二次确认后执行软删除
```

## 设计决策说明

### 导航设计

| 设计项 | 方案 | 理由 |
|--------|------|------|
| 导航形式 | 左边栏固定导航 | 符合国内 SaaS 产品习惯，扩展性好 |
| 菜单层级 | 二级菜单 | 销售管理 → 线索管理，便于后续扩展 |
| 当前模块 | 仅线索管理 | 聚焦核心需求，后续扩展客户/商机 |

### 简化设计

| 原竞品设计 | 简化方案 | 理由 |
|------------|----------|------|
| SuiteCRM 地址字段（省/市/区拆分） | 单行地址 | 减少表单复杂度 |
| SuiteCRM 传真字段 | 移除 | 国内场景不常用 |
| SuiteCRM 多地址类型 | 单一地址 | 小团队不需要 |
| 顶部导航栏 | 左边栏 + 二级菜单 | 更易扩展，符合国内习惯 |

### 借鉴设计

| 借鉴来源 | 功能 | 实现方式 |
|----------|------|----------|
| Twenty | 实时字段验证 | 邮箱/手机号失焦验证 |
| Twenty | 筛选器 URL 同步 | 筛选条件改变自动刷新 |
| SuiteCRM | 线索转换流程 | 弹窗式向导，支持多选 |
| SuiteCRM | 批量操作栏 | 底部固定操作栏 |
