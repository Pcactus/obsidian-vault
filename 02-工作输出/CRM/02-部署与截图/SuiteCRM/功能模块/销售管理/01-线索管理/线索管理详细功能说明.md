## 1. 功能概述

### 1.1 核心价值

**一句话说明**：线索管理是 CRM 系统的入口模块，管理从"潜在客户"到"成交客户"的完整转化流程。

**核心价值**：
- 集中管理所有潜在客户信息
- 跟踪线索来源和转化效果
- 支持线索分配和团队协作
- 提供线索转客户/联系人/商机的转换机制

### 1.2 适用场景

| 场景 | 说明 | 典型用户 |
|------|------|----------|
| 新线索录入 | 官网咨询、电话来电、活动收集的新客户 | 销售、市场人员 |
| 日常跟进 | 查看待跟进线索、更新沟通记录 | 销售人员 |
| 线索分配 | 主管将线索分配给团队成员 | 销售主管 |
| 线索转化 | 确认购买意向后转为正式客户 | 销售人员 |

### 1.3 前置条件

| 条件 | 说明 |
|------|------|
| 已登录系统 | 需要有效账号密码 |
| 拥有线索模块访问权限 | 管理员分配的基础权限 |
| 拥有线索编辑/分配权限 | 部分操作需要额外权限 |

## 3. 详细操作步骤

### 3.1 线索列表

**一句话说明**：查看所有线索的列表页，支持筛选、搜索、批量操作。

**入口**：主导航 → 销售 → 线索

**截图**：
![[销售管理_线索列表.png]]

**操作步骤**：
1. 点击主导航"销售"
2. 点击子菜单"线索"
3. 进入列表页

**页面元素**：

| 元素 | 说明 | 怎么用 |
|------|------|--------|
| 搜索框 | 按姓名/公司搜索 | 输入关键词后点搜索 |
| 状态筛选 | 按线索状态过滤 | 下拉选择后自动刷新 |
| 来源筛选 | 按线索来源过滤 | 下拉选择后自动刷新 |
| 负责人筛选 | 按负责人过滤 | 下拉选择后自动刷新 |
| 创建按钮 | 新建线索 | 点击右上角"创建线索" |
| 导入按钮 | 批量导入线索 | 点击右上角"导入" |
| 姓名列 | 线索姓名 | 点击进入详情页 |
| 操作列 | 更多操作 | 点击展开编辑/删除/分配 |
| 复选框 | 批量操作选择 | 勾选后激活批量操作 |
| 分页器 | 页码导航 | 点击页码跳转 |

**交互逻辑**：
- 筛选条件改变时，列表自动刷新（无需点击按钮）
- 点击姓名列链接，打开线索详情页
- 勾选复选框后，批量操作按钮激活
- 列表支持列排序，点击列标题切换升序/降序

**分析建议**：
- ✅ 保留：筛选器自动刷新体验好
- ✅ 保留：批量操作提升效率
- ⚠️ 简化：地址字段国内场景不需要省市区拆分

---

### 3.2 创建线索

**一句话说明**：录入新的潜在客户信息。

**入口**：线索列表 → 右上角"创建线索"按钮

**截图**：
![[销售管理_创建线索.png]]

**操作步骤**：
1. 点击右上角"创建线索"
2. 填写必填字段（姓、名）
3. 填写联系方式（可选）
4. 填写公司信息（可选）
5. 填写线索信息（来源、状态等）
6. 点击"保存"

**字段清单**：

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 姓 | 是 | 文本 | 最大 50 字符 |
| 名 | 是 | 文本 | 最大 50 字符 |
| 职位 | 否 | 文本 | 职务/头衔 |
| 部门 | 否 | 文本 | 所属部门 |
| 主要邮箱 | 否 | 邮箱 | 需格式校验 |
| 办公电话 | 否 | 电话 | 需格式校验 |
| 手机 | 否 | 电话 | 需格式校验 |
| 传真 | 否 | 电话 | 需格式校验 |
| 公司名称 | 否 | 文本 | 最大 100 字符 |
| 公司网址 | 否 | URL | 需格式校验 |
| 所属行业 | 否 | 下拉 | 预设行业列表 |
| 线索来源 | 否 | 下拉 | 网站/电话/推荐等 |
| 线索状态 | 是 | 下拉 | 默认"新建" |
| 线索评级 | 否 | 下拉 | Hot/Warm/Cold |
| 负责人 | 否 | 用户选择 | 默认当前用户 |
| 地址 | 否 | 多字段 | 省/市/区/街道/邮编 |
| 描述 | 否 | 文本域 | 备注信息 |

**交互逻辑**：
- 必填字段为空时，保存按钮置灰或点击后提示
- 邮箱、电话、URL 字段输入时实时格式校验
- 保存成功后，跳转到线索详情页
- 取消操作返回线索列表

**分析建议**：
- ✅ 保留：必填字段实时校验
- ⚠️ 简化：传真字段可省略
- ⚠️ 简化：地址字段简化为 1 行

### 3.3 线索详情

**一句话说明**：查看线索完整信息，包含基本信息、联系方式、关联记录和活动历史。

**入口**：线索列表 → 点击姓名

**截图**：
![[销售管理_线索详情.png]]

**操作步骤**：
1. 在线索列表页点击线索姓名
2. 进入详情页查看完整信息
3. 可进行编辑/转换/删除操作

**页面布局**：
- 顶部：基本信息展示区 + 操作按钮（编辑/转换/复制/删除）
- 中部：联系方式、公司信息、线索信息
- 底部：关联记录 + 活动历史

**关联记录**：
- 关联客户：线索转换后自动创建
- 关联联系人：线索转换后可创建
- 关联商机：线索转换后可创建
- 关联工单：手动或自动关联
- 关联会议/任务：日程管理中创建

**活动历史**：
- 邮件往来、电话记录、会议记录、任务记录、系统日志

**分析建议**：
- ✅ 保留：关联记录集中展示，信息完整
- ✅ 保留：活动历史时间线清晰
- ⚠️ 简化：活动历史初期仅记录系统日志

### 3.4 编辑线索

**一句话说明**：更新线索的任意字段信息。

**入口**：
- 线索列表 → 操作列 → 编辑
- 线索详情 → 顶部"编辑"按钮

**截图**：
![[销售管理_编辑线索.png]]

**操作步骤**：
1. 进入编辑页面（表单同创建页面）
2. 修改需要更新的字段
3. 点击"保存"

**交互逻辑**：
- 所有字段可编辑
- 修改后自动记录修改时间
- 保存后返回详情页

---

### 3.5 线索转换

**一句话说明**：将线索转为正式客户、联系人和商机，完成销售流程闭环。

**入口**：线索详情 → 右上角"转换"按钮

**截图**：
![[销售管理_线索转换.png]]

**操作步骤**：
1. 打开线索详情页
2. 点击右上角"转换"按钮
3. 选择转换目标：
   - 转为客户（必选）
   - 转为联系人（可选）
   - 转为商机（可选）
4. 确认数据映射
5. 选择原线索处理方式（删除/保留）
6. 点击"确认转换"
7. 跳转到新客户详情页

**转换后效果**：
- 新客户创建（必选）
- 新联系人创建（可选）
- 新商机创建（可选）
- 原线索状态变更为"已转换"
- 原线索关联到新创建的客户/联系人/商机

**分析建议**：
- ✅ 保留：转换流程灵活，支持多种组合
- ✅ 保留：可自选删除或保留原线索
- ⚠️ 简化：商机信息可简化（初期只需名称和金额）

### 3.6 批量操作

**一句话说明**：对多条线索进行批量分配、删除、导出等操作。

**入口**：线索列表 → 勾选线索 → 批量操作按钮

**截图**：
![[销售管理_线索列表.png]]

**支持的批量操作**：

| 操作 | 说明 | 注意事项 |
|------|------|----------|
| 批量分配 | 将选中的线索分配给指定用户 | 需要选择目标负责人 |
| 批量删除 | 软删除选中的线索 | 弹出确认框，防止误删 |
| 批量导出 | 导出选中的线索为 CSV | 选择导出字段 |
| 批量转状态 | 批量更新线索状态 | 选择目标状态 |

**分析建议**：
- ✅ 保留：批量分配提升团队管理效率
- ✅ 保留：批量导出方便数据备份
- ⚠️ 建议增强：增加批量导入功能（支持 Excel）

### 3.7 线索导入

**一句话说明**：从 CSV/Excel 文件批量导入线索数据。

## 4. 数据流转说明

### 4.1 线索表结构

```sql
CREATE TABLE leads (
    -- 基础信息
    id              VARCHAR(36) PRIMARY KEY,
    deleted         TINYINT(1) DEFAULT 0,
    date_entered    DATETIME DEFAULT CURRENT_TIMESTAMP,
    date_modified   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 个人信息
    first_name      VARCHAR(50),          -- 姓
    last_name       VARCHAR(50),          -- 名
    title           VARCHAR(50),          -- 职位
    department      VARCHAR(50),          -- 部门
    
    -- 联系方式
    email1          VARCHAR(100),         -- 主要邮箱
    phone1          VARCHAR(20),          -- 办公电话
    phone2          VARCHAR(20),          -- 手机
    
    -- 地址
    street          VARCHAR(255),
    city            VARCHAR(50),
    state           VARCHAR(50),
    postalcode      VARCHAR(20),
    country         VARCHAR(50),
    
    -- 公司信息
    company         VARCHAR(100),         -- 公司名称
    website         VARCHAR(255),         -- 公司网址
    industry        VARCHAR(50),          -- 所属行业
    
    -- 线索信息
    lead_source     VARCHAR(50),          -- 来源
    lead_rating     VARCHAR(20),          -- 评级 (Hot/Warm/Cold)
    status          VARCHAR(50),          -- 状态
    assigned_user_id VARCHAR(36),         -- 负责人
    
    -- 描述
    description     TEXT,
    
    -- 索引
    INDEX idx_status (status),
    INDEX idx_assigned (assigned_user_id),
    INDEX idx_company (company)
);
```

### 4.2 线索状态枚举

| 状态 | 说明 |
|------|------|
| 新建 (New) | 刚创建的线索，尚未联系 |
| 已联系 (Contacted) | 已取得联系，正在跟进 |
| 有意向 (Interested) | 明确表示有兴趣 |
| 已评估 (Qualified) | 已评估，符合目标客户标准 |
| 未评估 (Unqualified) | 已评估，不符合目标客户标准 |
| 已转换 (Converted) | 已转为客户/联系人/商机 |
| 已关闭 (Closed) | 已关闭（放弃或完成） |

## 5. 自研 CRM 建议

### 5.1 建议保留的功能（P0/P1）

| 功能 | 优先级 | 理由 |
|------|--------|------|
| 线索 CRUD | P0 | 核心基础功能 |
| 线索状态管理 | P0 | 销售流程跟踪必需 |
| 线索来源跟踪 | P0 | 营销效果分析基础 |
| 线索搜索/筛选 | P0 | 高频使用功能 |
| 线索转换 | P1 | 销售流程闭环 |
| 批量操作 | P1 | 提升效率 |

### 5.2 建议简化的功能

| 功能 | 简化方案 | 理由 |
|------|----------|------|
| 地址字段 | 简化为 1 行 | 国内场景不需要复杂的省市区拆分 |
| 传真 | 可省略 | 现代业务很少使用 |
| 团队管理 | 初期可省略 | 小团队不需要复杂权限 |
| 线索评级 | 简化为高/中/低 | 不需要太复杂的评级体系 |
| 活动历史 | 初期仅记录系统日志 | 邮件/电话集成可后续添加 |

### 5.3 建议增强的功能

| 功能 | 增强方向 | 理由 |
|------|----------|------|
| 线索分配 | 支持自动分配规则 | 提高团队效率 |
| 线索去重 | 创建时自动检测重复 | 避免数据冗余 |
| 线索导入 | 支持 Excel 导入 | 符合国内用户习惯 |
| 移动端适配 | 响应式设计 | 移动办公需求 |
