## 2 快速开始

### 2.1 常用命令

| 场景 | 命令 | 说明 |
|------|------|------|
| 启动新分析 | `/investigate` | 系统化调研未知领域 |
| 代码审查 | `/review` | 提交前代码审查 |
| 网页浏览 | `/browse <URL>` | 所有网页任务统一用此 |
| 部署上线 | `/land-and-deploy` | 自动化部署流程 |
| 质量测试 | `/qa` | 系统化测试并修复 |

### 2.2 ⚠️ 重要提醒

> [!warning] 网页浏览约束
> **必须使用** `/browse` 技能进行所有网页浏览任务
> **禁止使用** `mcp__claude-in-chrome__*` 工具

## 5 文件命名约定

### 5.1 报告文件

```
[产品类型]-战略规划报告 -[YYYYMMDD].md
[项目名]-部署报告.md
[项目对比总结].md
[产品名称]-PRD-v1.0.md              # Phase 2
```

### 5.2 截图文件

```
[项目名]_[页面路径]_[YYYYMMDD].png
```

示例：`Plane_auth_20260408.png`

### 5.3 日记/笔记

```
YYYY-MM-DD.md          # 日记
[主题]-[子主题].md      # 普通笔记
```

## 6 路径处理规范

| 场景 | 路径类型 | 示例 |
|------|----------|------|
| 知识库文件 | 相对路径 | `00-索引/工作流总览.md` |
| 外部上下文 | 绝对路径 | `/Users/me/Workspace/file.ts` |
| 回复中引用 | Wikilink | `[[00-索引/工作流总览]]` |

> [!error] 错误示例
> - ❌ `/00-索引/工作流总览.md` (前导斜杠)
> - ❌ `/home/ubuntu/Documents/Obsidian Vault/file.md` (绝对路径访问知识库)
